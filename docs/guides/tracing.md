# Request Tracing

Vorte provides request tracing via context variables for tracking requests across the system.

## How It Works

Each request gets a unique trace ID generated by the middleware. This ID is stored in a `ContextVar` and included in the response headers.

```python
from vorte.core.tracing import get_trace_id, set_trace_id, generate_trace_id
```

## Functions

| Function | Description |
|----------|-------------|
| `get_trace_id() -> str` | Get the current trace ID |
| `set_trace_id(trace_id: str)` | Set a trace ID (returns reset token) |
| `reset_trace_id(token)` | Reset to previous trace ID |
| `generate_trace_id() -> str` | Generate `"req_"` + 16 hex chars |

## Usage in Route Handlers

```python
from vorte.core.tracing import get_trace_id

@app.get("/api/v1/data")
async def get_data():
    trace_id = get_trace_id()  # e.g., "req_a1b2c3d4e5f6g7h8"
    # Use for logging, distributed tracing, etc.
    logger.info(f"[{trace_id}] Fetching data")
    return success_response(data={...})
```

## Response Headers

The trace ID is automatically included in every response:

```http
HTTP/1.1 200 OK
X-Request-ID: req_a1b2c3d4e5f6
X-Powered-By: Vorte
X-Response-Time: 12.34ms
```

## Middleware Flow

The Vorte middleware sets up tracing for each request:

1. Generates a unique request ID
2. Sets the `trace_id_var` ContextVar
3. Processes the request
4. Adds `X-Request-ID` header to the response
5. Resets the ContextVar on exit

## Custom Trace ID Generation

```python
from vorte.core.tracing import generate_trace_id

custom_id = generate_trace_id()  # "req_" + 16 hex chars
```
